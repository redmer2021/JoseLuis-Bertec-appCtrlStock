Resolver cuando se quedó bloqueada una base de datos y no te deja restaurar
🧩 Paso 1: Conectarte a master

Abrí una nueva ventana de consulta en SSMS y elegí la base master, no BERTEC_SA.
Esto es importante, porque si estás conectado a la misma base que querés modificar, no te dejará hacerlo.

🧩 Paso 2: Identificar quién está conectado

Ejecutá esto:

SELECT 
    session_id,
    login_name,
    host_name,
    program_name
FROM sys.dm_exec_sessions
WHERE database_id = DB_ID('BERTEC_SA');


Esto te mostrará quién tiene la conexión activa (puede ser Management Studio, un servicio, etc.).

🧩 Paso 3: Terminar esa sesión

Supongamos que el resultado muestra un session_id = 57 (por ejemplo).
Entonces ejecutás:

KILL 57;


⚠️ Si aparecen varios session_id, tenés que matarlos todos.

🧩 Paso 4: Cambiar el modo de acceso

Una vez que la base quede sin conexiones activas, ejecutá:

ALTER DATABASE [BERTEC_SA] SET MULTI_USER WITH ROLLBACK IMMEDIATE;
